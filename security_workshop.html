Information

Passwords: It's (Almost) All Math

A password of any given length (L) built using a character set of size (N - which represents complexity) creates a combination of posibilities where:

Possible Combinations = N L
As the English alphabet contains 26 characters, if we have a password length of 8 alphabetical characters (only including lowercase letters) then the complexity becomes:

Possible Combinations = 26 8 = 208,827,064,576
The mathematical expression explains why password length and not simply complexity make the biggest impact. Complexity is the base and password length is the exponent.

The following table exemplifies how simply changing complexity will affect the possible combinations for a password with a length of 8 characters:

Characters (Length = 8) Complexity (N) Possible Combinations
Lowercase letters only 26 208,827,064,576
Lowercase letters + Uppercase letters 52 53,459,728,531,456
Lowercase + Uppercase letters + Numbers 62 218,340,105,584,896
All letters + Numbers + Special Characters ~95 6,634,204,312,890,625

The addition of complexity does cause the number of possibilities to increase, however, not to the same magnitude as when we increase the length of the password:

Lab 3

What to carry out the door

Passwords

  • Length wins. A long passphrase of ordinary words beats a short string of symbols. Aim for 16+ characters.
  • Never reuse. One breached site shouldn't unlock the rest of your life. A password manager makes unique passwords effortless.
  • Turn on multi-factor authentication. Even a cracked password is useless without the second factor.
  • Don't build passwords from public facts — names, mascots, the current year, the season. Attackers guess these first.

Email & phishing

  • Check the actual sender address, not just the display name. Hover before you trust.
  • Urgency is a weapon. "Act now or lose access" is designed to stop you thinking. Slow down.
  • Hover over links to see where they really go. When in doubt, navigate to the site yourself instead of clicking.
  • Verify money and data requests through a second channel — a phone call, an in-person check especially if the "sender" is a supervisor.
  • Report, don't just delete. Forwarding a phish to IT protects the colleague who gets it next.
